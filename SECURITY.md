# Security Policy

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please report it privately.

**Do not** disclose security vulnerabilities publicly via GitHub issues.

Instead, email security@codeeval.app or use GitHub's private vulnerability reporting.

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if applicable)

## Response Timeline

- Acknowledgment within 48 hours
- Initial assessment within 5 business days
- Fix timeline communicated based on severity

## Security Measures

- All communications over TLS 1.3
- Row-Level Security on all user data
- No execution of user-uploaded code
- Input validation on all API endpoints
- Secure HTTP headers (CSP, HSTS, etc.)
- Environment variables for all secrets
- Regular dependency audits
